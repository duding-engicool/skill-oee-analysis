# OEE 分析改善技能 · 交互引导

> 轻交互采集引导，目标是用最小提问拿到可计算的数据，不问卷、不代决策。

## 一、开场确认（先判定载体）

用户来咨询时，先问清一件事：**数据长什么样？**

- 投文件（Excel/CSV/MES 导出）→ 引导上传，技能读表提取
- 贴数字 / 描述 → 直接收
- 截图 / PDF 报表 → 引导转写关键数字

## 二、最小必需数据采集

只问这 4 项（缺则无法算 OEE）：

1. 计划运行时间（小时）
2. 实际运行时间（小时）
3. 总产量（件）
4. 合格品数量（件）

## 三、可选数据（越全诊断越准，不强求）

- 六大损失时间：故障 / 换模 / 空转 / 减速 / 启动 / 质量损失（小时）
- 标准节拍（秒/件）
- 设备每小时产值（元）

## 四、企业参数处理（默认值 + 标注）

| 参数 | 默认值 | 报告标注 |
|------|------|------|
| 标准节拍 | 60 秒/件 | 待企业补充 |
| 每小时产值 | 5000 元 | 待企业补充 |
| 年运行工时 | 2400 h | 待企业补充 |
| 目标 OEE | 85% | 待企业补充 |

用户给了就用真实值，没给就用默认并在报告显式标「待企业补充」。

## 五、针对性追问（不发长问卷）

- 若 OEE 显示极高但用户说现场常停线 → 直接点出"计划运行时间可能缩水"，请核实。
- 若只有 4 项必需数据 → 提示"损失构成无法拆解，建议补停机明细以定位改善重点"。
- 始终先说"我将基于你给的 X 数据计算 OEE"再动手。
