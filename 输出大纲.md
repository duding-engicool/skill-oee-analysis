# OEE 诊断报告 · 输出大纲（MD + HTML 双版）

> 生成报告前先给用户此大纲确认，再调用 `scripts/build_report.py` 装配双版。

## 模块结构

1. **基本信息**：设备 / 统计周期 / 企业（缺失标待补充）
2. **一、OEE 三大指标**：时间开动率 / 性能开动率 / 合格品率 / OEE，含目标对比与达标判定
3. **二、六大损失分布**：各损失占计划运行时间比，按大小排序，标最大损失项
4. **三、数据可信度检测**：预警清单（计划缩水/节拍放宽/口径不实等）+ 核实建议
5. **四、改善优先级建议**：优先方向 + 依据 + 当前→预估 OEE 区间 + 3 条措施
6. **五、ROI 估算**：每小时产值 / 年工时 / 目标 OEE / 可挽回产值 / 回收期，**所有假设标「默认值·待企业补充」**
7. **六、改善措施库参考**：三大短板维度措施速查

## 双版差异

| 版本 | 形态 | 适用 |
|------|------|------|
| MD 版 | 表格化文字 | 归档、复制进文档、轻量查看 |
| HTML 版 | 指标仪表 + 损失占比条形图（内联样式，离线可开） | 汇报演示、直观展示 |

## 输入 JSON 结构（供 build_report.py 调用）

```json
{
  "device": "设备A",
  "period": "2026年3月",
  "data": {
    "planned_time": 528, "actual_time": 458,
    "total_output": 18500, "qualified_output": 17945,
    "standard_cycle_time": 90,
    "downtime_failure": 32, "downtime_changover": 18,
    "idle_time": 8, "speed_loss_time": 6,
    "startup_time": 4, "quality_loss_time": 2,
    "hourly_output_value": 5000
  },
  "enterprise": {"company": "待企业补充", "annual_hours": "待企业补充", "target_oee": "待企业补充"},
  "notes": "",
  "_provided": {"hourly_output_value": true}
}
```

> `_provided` 标记用户真实提供的企业参数，用于报告区分"真实值"与"默认值·待企业补充"。
