# 改善措施库（中性实战版）

> 本文件为 Agent 给出改善建议时的措施候选库，按三大短板维度组织。所有措施均为通用管理实践，落地需结合企业现场条件。

## 一、时间开动率短板（可用性损失）

| 措施 | 方法要点 |
|------|------|
| 设备故障减少 | 建立 MTBF/MTTR 追踪体系，将平均故障处理时间控制在 2 小时以内 |
| 换模效率提升 | 导入快速换模(SMED)法，将换型时间缩短 50% |
| 设备保养闭环 | 每周定保全点检，小问题当天清不过夜，建立隐患台账 |

## 二、性能开动率短板（性能损失）

| 措施 | 方法要点 |
|------|------|
| 消除空转暂停 | 分析待料/断料/堵料原因，建立物料预警与齐套性管理机制 |
| 解决减速损失 | 排查设备老化降速根因，优化工艺参数与设备能力 |
| 换模效率翻倍 | 用快速换模法分解换型步骤，外部作业与内部作业并行 |

## 三、合格品率短板（质量损失）

| 措施 | 方法要点 |
|------|------|
| 降低不良率 | 分析不良类型分布，针对性改善关键工艺控制点 |
| 减少返工品 | 建立首件确认机制，减少批量质量事故 |
| 异常响应加速 | 安灯一响，10 分钟内到场，30 分钟内给出处置方案 |

## 四、使用说明

- 改善优先级由引擎按「当前最大短板」自动判定（见 six-losses-classification.md）。
- 报告给出的「改善后预估 OEE」仅为基于 +5pt / +1pt 简化的示意区间，**非承诺值**，须标注为参考。
- ROI 估算依赖企业工时、产值、目标等参数，缺失时一律标注「待企业补充」，不填默认值当结论。
